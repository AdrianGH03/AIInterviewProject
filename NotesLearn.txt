Q What does this line mean and what does each library do and for what? "pip install fastapi uvicorn sqlalchemy psycopg2-binary alembic pydantic python-dotenv celery redis httpx"
- fastapi: A modern, fast (high-performance) web framework for building APIs with Python 3.6+ based on standard Python type hints.
- uvicorn: A lightning-fast ASGI server implementation, using uvloop and httptools. (ASGI is a specification for Python web servers and applications to communicate with each other)
- sqlalchemy: A SQL toolkit and Object-Relational Mapping (ORM) library for Python. It provides a high-level interface for working with databases, allowing developers to interact with databases using Python objects instead of writing raw SQL queries.
- psycopg2-binary: A PostgreSQL adapter for Python. It allows Python code to interact with a PostgreSQL database, enabling you to execute SQL commands and manage database connections.
- alembic: A database migration tool for SQLAlchemy. It helps manage database schema changes over time, allowing you to create, modify, and maintain your database schema in a version-controlled manner.
- pydantic: A data validation and settings management library for Python. It uses Python type annotations to validate and parse data, making it easier to work with complex data structures and ensuring that the data conforms to the expected types.
- python-dotenv: A library that allows you to read key-value pairs from a .env file and set them as environment variables. This is useful for managing configuration settings and secrets in a secure and organized way.
- celery: An asynchronous task queue/job queue based on distributed message passing. It is focused on real-time operation but also supports scheduling. Celery is used to execute tasks in the background, allowing for better performance and responsiveness in web applications.
- redis: An in-memory data structure store, used as a database, cache, and message broker. In the context of Celery, Redis is often used as a message broker to manage task queues.
- httpx: A fully featured HTTP client for Python, which provides a simple and intuitive API for making HTTP requests. It supports features like connection pooling, HTTP/2, and asynchronous requests, making it a powerful tool for interacting with web services and APIs.

Q What is circular dependency and how do you solve it?
- Circular dependency occurs when two or more modules depend on each other, creating a loop that can lead to import errors. To solve it, you can:
  - Refactor the code to remove the circular dependency by moving shared functionality to a separate module.
  - Use local imports within functions instead of global imports at the top of the file to delay the import until it's needed.
  - Use dependency injection to pass instances of classes instead of importing them directly.
